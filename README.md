# Mittare Sattahip + MITTARE Exam Coach

โปรเจกต์นี้รวมเว็บไซต์บริการประกันภัย Mittare Sattahip และระบบฝึกข้อสอบไว้บนเว็บเดียวกัน:

- `/` เว็บไซต์หลัก ผลิตภัณฑ์ เช็กสถานะ และหลังบ้านตัวแทน
- `/exam/` ระบบฝึกข้อสอบ
- `/exam/admin` ระบบจัดการคลังข้อสอบ

ฐานข้อมูลลูกค้าอยู่ที่ `APP_INSTANCE_DIR/mittare.sqlite3` ส่วนฐานข้อมูลข้อสอบกำหนดด้วย `DATABASE_PATH` จึงสำรองและย้ายข้อมูลแยกกันได้

ระบบฝึกข้อสอบมีสมาชิกแบบง่ายด้วยชื่อผู้ใช้และรหัสผ่าน รหัสผ่านถูก hash ก่อนบันทึก คะแนนและสถิติผูกกับบัญชีแต่ละคนใน SQLite ส่วนทางเข้าจัดการคลังข้อสอบแสดงภายในหลังบ้านตัวแทนหลังเข้าสู่ระบบแล้ว

เว็บฝึกสอบนายหน้าประกันวินาศภัย Flask + SQLite สำหรับส่งให้ทีมเปิดผ่านโทรศัพท์หรือคอมพิวเตอร์

## ใช้งาน

Windows: ดับเบิลคลิก `run_server.bat` ระบบจะสร้าง Python environment ติดตั้ง Flask สร้างฐานข้อมูล และเปิดเว็บที่ `http://127.0.0.1:5000` อัตโนมัติ

หรือรันเองด้วย `pip install -r requirements.txt` แล้ว `python server.py`

เว็บจะสุ่ม 20 ข้อจากคลัง 120 ข้อใน SQLite ต่อรอบ สลับลำดับตัวเลือก และเก็บคะแนนกับรายการข้อสอบรอบล่าสุดใน `localStorage` ของเบราว์เซอร์เพื่อลดข้อซ้ำในรอบถัดไป ฐานข้อมูลจะถูกสร้างที่ `data/exam_coach.sqlite3` อัตโนมัติ

ระบบรองรับผู้ฝึกหลายคนด้วยชื่อเล่น บันทึกผลแต่ละรอบลง SQLite กลาง แสดงอันดับทีม และเลือกฝึกแยกตามหมวดได้ คลังปัจจุบันเป็นแนวข้อสอบที่เรียบเรียงตามหัวข้อหลักสูตร คปภ. ไม่ได้กล่าวอ้างว่าเป็นข้อสอบเก่าที่ คปภ. เผยแพร่หรือรับรอง

## การปรับข้อสอบ

เปิด `/admin` เพื่อจัดการคลังข้อสอบ ระบบรองรับบัญชี `head` สำหรับสร้าง/แก้ไข/ส่งตรวจ และ `admin` สำหรับอนุมัติ ส่งกลับ ระงับ กู้คืน จัดการบัญชี และดู Audit Log ข้อสอบที่แก้ไขจะกลับเป็นฉบับร่างเสมอ และจะเข้าสู่หน้าฝึกเฉพาะสถานะ `published` เท่านั้น

ตั้งค่าตัวแปรแวดล้อมก่อนเปิดใช้งานครั้งแรก:

```text
SECRET_KEY=ค่าสุ่มที่ยาวและเก็บเป็นความลับ
ADMIN_USERNAME=mt4admin
ADMIN_PASSWORD=รหัสผ่านเริ่มต้นอย่างน้อย 12 ตัว
RESET_ADMIN_PASSWORD=0  # เปลี่ยนเป็น 1 ชั่วคราวเมื่อต้องการรีเซ็ตรหัส Admin เดิม
COOKIE_SECURE=0  # ใช้เฉพาะ localhost ที่ไม่มี HTTPS
BACKUP_ENCRYPTION_KEY=URL-safe base64 ของค่าสุ่ม 32 bytes
```

ระบบจะสร้างบัญชี Admin เริ่มต้นจากตัวแปรเหล่านี้เฉพาะเมื่อฐานข้อมูลยังไม่มีผู้ดูแล หลังเข้าสู่ระบบแล้ว Admin สามารถสร้างบัญชีรายบุคคลให้หัวหน้าทีมได้ รหัสผ่านเก็บเป็น hash และคำสั่งแก้ไขใช้ session cookie, CSRF token, version history และ audit log

หากใช้ Persistent Database และลืมรหัสเดิม ให้ตั้ง `ADMIN_USERNAME` เป็นบัญชีที่ต้องการ ตั้ง `ADMIN_PASSWORD` เป็นรหัสใหม่ และตั้ง `RESET_ADMIN_PASSWORD=1` แล้ว deploy หนึ่งครั้ง ระบบจะเปลี่ยนรหัสและปลดล็อกการเข้าสู่ระบบของบัญชีนั้น หลังเข้าสู่ระบบสำเร็จต้องเปลี่ยน `RESET_ADMIN_PASSWORD` กลับเป็น `0` เพื่อไม่ให้รีเซ็ตซ้ำเมื่อเซิร์ฟเวอร์เริ่มใหม่

คลังเดิมข้อ 1-50 อยู่ใน `app.js` และข้อ 51-120 อยู่ใน `extra_questions.py` เพื่อ seed ฐานข้อมูลเฉพาะรายการที่ยังไม่มี ระบบจะไม่เขียนทับข้อสอบที่แก้ผ่านหลังบ้านเมื่อเริ่มเซิร์ฟเวอร์

> เนื้อหากฎหมายและวงเงินอาจมีการแก้ไข ควรตรวจประกาศและแบบกรมธรรม์ล่าสุดของ คปภ. ก่อนนำไปใช้อ้างอิงจริง

## Deploy บน Render

โปรเจกต์มี `render.yaml` พร้อมใช้งาน: เชื่อม GitHub repository ในหน้า **New > Blueprint** ของ Render แล้วเลือก repository นี้ ระบบจะติดตั้งแพ็กเกจ รันด้วย Gunicorn และตรวจสุขภาพผ่าน `/healthz` อัตโนมัติ

โลโก้โปร่งใสทั้งสามอยู่ใน `assets/mt4-heart-logo.png`, `assets/team-community-logo.png` และ `assets/one-team-logo.png` โดยใช้แยกกันคนละตำแหน่งบนหน้าเว็บ

`render.yaml` สร้าง `SECRET_KEY` อัตโนมัติและกำหนดให้กรอก `ADMIN_PASSWORD` เป็น secret ใน Render Dashboard

Full Backup ถูกเข้ารหัสแบบ AES-256-GCM และดาวน์โหลดเป็น `.mtbackup` ระบบจะปิดการดาวน์โหลดแบบ fail-safe หากยังไม่ตั้ง `BACKUP_ENCRYPTION_KEY` วิธีสร้างกุญแจและถอดรหัสอยู่ใน `SECURITY.md`

> สำคัญ: แผน Free ของ Render ใช้ filesystem ชั่วคราว ข้อมูลที่เพิ่มผ่านหลังบ้าน บัญชีผู้ดูแล และคะแนนส่วนกลางอาจหายเมื่อ deploy/restart ก่อนใช้งานจริงต้องเพิ่ม Render Persistent Disk แล้วเปลี่ยน `DATABASE_PATH` เป็น path บน disk (ใช้ web service หนึ่ง instance) หรือย้ายไป Render Postgres การเพิ่ม Persistent Disk อาจมีค่าใช้จ่ายจึงไม่ได้เปิดให้อัตโนมัติจาก repository
